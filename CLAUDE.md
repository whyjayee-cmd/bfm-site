# BfM 사이트 — 프로젝트 컨텍스트 (Claude용)

이 파일은 Claude Code가 자동으로 읽는 프로젝트 설명서입니다. 디자이너가
이 폴더에서 Claude Code를 켜면, 별도로 설명하지 않아도 아래 내용을 맥락으로
가지고 작업을 시작합니다.

## 이 프로젝트가 뭔가
BfM(Bespoke For Man) — 남성 대상 **AI 피부진단 → 맞춤 처방** 랜딩/서비스 사이트.
순수 정적 웹사이트이며 **GitHub Pages**로 배포됩니다.

## 기술 스택 (중요: 빌드 과정이 없음)
- **빌드 도구·프레임워크·npm 없음.** HTML 파일을 그대로 브라우저로 열면 동작합니다.
- 각 HTML 파일은 **자기 안에 `<style>`(CSS)와 `<script>`(JS)를 전부 포함**한 독립 파일입니다.
  공유 .css / .js 파일이 따로 없습니다.
- 외부 의존성은 CDN 2개뿐:
  - **Pretendard** 폰트 (jsdelivr CDN)
  - **MediaPipe tasks-vision** — `diagnose.html`의 얼굴 감지/랜드마크용 (jsdelivr + Google CDN)

## 파일 구조 (이게 전부입니다)
| 파일 | 역할 |
|---|---|
| `index.html`    | PC 메인 랜딩 페이지 |
| `m.html`        | 모바일 랜딩 페이지 |
| `diagnose.html` | AI 피부진단 — 카메라/사진 + MediaPipe 얼굴인식. 가장 복잡한 로직 |
| `survey.html`   | 피부 진단 설문 |
| `result.html`   | 진단 후 맞춤 처방 결과 |

**사용자 흐름:** 랜딩(`index.html` / `m.html`) → `diagnose.html` → `result.html`

## ⚠️ 디자인 작업 시 반드시 알아야 할 점
- **디자인 토큰(색상·반경·폰트 등 CSS 변수)이 5개 파일의 `:root`에 각각 복붙되어 있습니다.**
  예: `--accent:#5B8DEF`(메인 블루), `--bg:#0B0C0E`(배경), `--ink:#EDEFF2`(글자색).
  → "메인 색을 바꿔줘" 같은 요청은 **5개 파일을 모두** 일관되게 수정해야 합니다.
  한 파일만 고치면 페이지마다 색이 달라집니다.
- 다크 테마 기반 + 일부 "클리닉 화이트"(`--clinic*`) 섹션이 섞여 있습니다.
- 모바일은 `m.html`이 따로 있으니, 모바일 변경은 `m.html`도 함께 확인하세요.

## 개인정보 / 안전
- 얼굴인식은 **전부 브라우저(클라이언트)에서** 실행됩니다. 사진/얼굴 데이터를
  서버로 보내지 않습니다. 이 원칙을 깨는 변경(이미지 업로드/전송 추가 등)은
  넣기 전에 반드시 사용자에게 확인하세요.

## 로컬에서 보는 법
빌드가 없으므로 둘 중 아무거나:
- 파일을 브라우저로 바로 열기 (`index.html` 더블클릭)
- 또는 간단한 정적 서버: `python3 -m http.server 8000` 후 `http://localhost:8000`
  (얼굴인식 카메라 기능은 `file://`보다 `localhost`에서 더 잘 동작합니다)

## 작업 규칙
- 변경 후에는 실제로 브라우저에서 해당 페이지를 열어 깨지지 않는지 확인.
- 색/폰트/간격 등 토큰성 변경은 5개 파일 전체 일관성 유지.
- 커밋 메시지는 무엇을 왜 바꿨는지 한국어로 명확하게.
